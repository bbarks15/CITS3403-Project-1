CSS

Cascading Style Sheet

History

HTML and CSS (Cascading Style Sheet) often go hand in hand. However, both are very different in the function they provide to constructing pages on the web. HTML is used to generate the content, whilst CSS is used for styling that content. HTML is analogous to the construction of the house, whilst CSS is analogous to the decoration. HTML and CSS did not always co-exist. CSS was invented many years after its counterpart. When HTML was invented in late 1990, there was no way to style a website. In 1994 Marc Andreessen wrote a now-infamous when asked about visual design and styling a website with HTML. In the post he responded with

In fact, it has been a constant source of delight for me over the past year to get to continually tell hordes (literally) of people who want to -- strap yourselves in, here it comes -- control what their documents look like in ways that would be trivial in TeX, Microsoft Word, and every other common text processing environment: "Sorry, you're screwed."

Marc Andreessen 1994 post

Style sheets have existed since the beginnings of SGML (Standard Generalized Markup Language) in the 1980's. However, these implementations did not allow for style sheets to come from different sources on the web. This became a massive turn off for existing style sheets. As HTML grew, it added functionality for styling. This gave web developers more control over the style, but it came at the cost increasing the complexity of HTML. The stark differences in web browser implementations, such as ViolaWWW and WorldWideWeb, made consistent styling difficult. As users had less control over how the content would be displayed in the browser. For example the NeXT browser made by Sir Tim Berners-Lee had style sheets hard coded into the program. Thus, the style sheets could not be linked to documents on the web. This spawned the idea of separating structure and style. Enabling an external style sheet to describe the style of page, allowing for more control and consistency. On October 10, 1994 Hakon Wium Lie, a colleague of Sir Tim Berners-Lee proposed the idea of CSS. CSS or cascading style sheets let a document's style be influenced by multiple style sheets by its ability for style to cascade. Cascade in the context of CSS means that style can inherit and overwrite other styles that have already been defined. Thus, the name CSS.

HTML is meant for structure, not anything else. Having the styling outside of HTML helps to keep it concise to its original purpose. Instead of it becoming an amalgamation of what it was intended to be. The benefits of CSS are

Maintainability and Complexity Consistency Bandwidth Reduction Cascading Control More Control Even More Control

Keeping the style and structure separate, improves readability and makes it easier to make changes.

Modularising of the style reduces the complexity. This makes it easier to edit and update. Instead of having to search through a large document with both style and content. You only need to go to the one you want to change

CSS allows for single style sheet to style multiple documents. This document has a general style sheet, that applies to all the pages and individual style sheets that apply to a specific page.

An example of global styling is shown with the body tag. 

body {
    font-family: "Open Sans", sans-serif;
    font-size: 14pt;
    ...
}

This styling makes the font family and font size the same throughout all the documents. In the individual style sheets colors can be specified. For example the list to the left has the color Blue for active elements. This only applies to css page, as other pages have different colors specified in their style sheets. 

.list-group-item.active {
    color: #212529;
    background-color: #3399FF;
    border-color: #3399FF;
}

This is more a of niche benefit, but separating the style from the HTML document largely reduces the file transfer size.

The external style sheet will only be accessed when someone requests the website. The style sheet is cached and only needs to be accessed once. Improving performance.

The cascading in CSS is a very important feature. Cascading in CSS is the ability for conflicting styles to be prioritized, and for properties to be inherited from their parents.

The general priority of properties are

  1. Inline style (inside HTML element)
  2. External and internal stylesheets
  3. Browser defaults

The order properties are defined is also very important. So for example linking one style sheet below another sheet, gives the sheet below higher priority. Linking the user written style sheet after the bootstrap style sheet allows the user to override, and customise the bootstrap styles.

CSS uses selectors to specify what element is to be styled. The main three selectors are by class, id or element. The styling for the body tag is an example of element styling. This styling will apply to all body elements.

Classes can be defined by the developer. Essentially creating your own grouping of elements to have a specific styling. Classes are denoted by a "." before the name of the class in CSS. Here is an example 

.exampleCode {
    margin-left: auto;
    margin-right: auto;
    padding-top: 1cm;
    padding-bottom: 1cm;
    width: 70%;
}

The exampleCode class gives all elements with the class exampleCode the following styling. Lastly is id. An id is like a class, but it only applies to one element. In CSS an id is denoted by the "#" symbol 

#CSS_Logo_Text {
    font-size: 32pt;
}

This only applies to the element with the CSS_Logo_Text id

The selectors defined in the previous section can be combined with "operators" to give even more control of the styling of elements

Descendent selector

a b {
    ...
}

The descendant selector matches all elements that are descendants of a specified element.

Child selector (a > b)

a > b {
    ...
}

The child selector selects all elements that are the children of a specified element.

Adjacent sibling selector (a + b)

a + b {
    ...
}

The adjacent sibling selector selects all elements that are the adjacent siblings of a specified element.

General sibling selector (a ~ b)

a + b {
    ...
}

The general sibling selector selects all elements that are siblings of a specified element. These operators can be combined together for example

.prettyprint ol.linenums > li {
    list-style-type: decimal;
}

This only applies to <li> elements that are children of elements with the prettyprint class and ol.linenums , <ol> elements with the linenums classes.

CSS also has pseudo classes. A pseudo class is used to define the state of an element 

selector:pseudo-class {
    ...
}

Here are some of the pseudo-classes to give an idea of what is possible

  • :active
  • ::after and ::before
  • :hover
  • :first-child
  • :nth-child

For example, in the timeline on the HTML page. The timeline uses the :nth-child() pseudo class to specify different colors for the nth recurring icon on the timeline. Below is the code 

.main-timeline .timeline:nth-child(2n) .timeline-icon {
    border-color: #7eda99
}
